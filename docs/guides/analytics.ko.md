# 분석 (Analyze) 가이드

데이터 기반의 의사결정을 지원하기 위한 시스템 모니터링 및 분석 기능을 설명합니다.

## 1. 대시보드 (Dashboard)

시스템의 전체 현황을 한눈에 파악할 수 있는 메인 화면입니다.

### 주요 지표

- **총 사용량**: 오늘의 총 토큰 사용량과 요청 수.
- **활성 사용자**: 현재 접속 중이거나 오늘 AI를 사용한 사용자 수.
- **인기 모델**: 가장 많이 사용된 AI 모델 순위.
- **오류율**: 전체 요청 대비 실패한 요청의 비율.

## 2. 품질 분석 (Quality)

AI 응답의 품질을 지속적으로 관리하고 개선하기 위한 도구입니다.

### 기능

- **사용자 피드백**: 채팅 메시지에 대한 좋아요/싫어요 및 피드백 내용을 수집하여 분석합니다.
- **자동 평가**: LLM을 사용하여 응답의 정확성, 관련성, 안전성을 자동으로 점수화합니다.
- **이슈 탐지**: 낮은 점수를 받거나 부정적인 피드백이 많은 대화를 식별하여 검토할 수 있게 합니다.

## 3. 비용 분석 (Cost)

AI 운영 비용을 추적하고 최적화합니다.

### 기능

- **모델별 비용**: 각 모델(GPT-4, Claude 3 등)이 소모한 비용을 비교합니다.
- **부서별/사용자별 비용**: 누가, 어느 부서에서 예산을 많이 사용하고 있는지 파악합니다.
- **예산 알림**: 설정된 월 예산에 도달하면 관리자에게 알림을 보냅니다.

## 4. ML 운영 (MLOps)

자체 호스팅 모델이나 미세 조정(Fine-tuning)된 모델의 수명 주기를 관리합니다.

### 기능

- **배포 상태**: 내부 모델 서버(vLLM, Ollama)의 상태(Running, Stopped, Error)를 모니터링합니다.
- **모델 버전 관리**: 모델의 업데이트 이력과 버전을 관리하고, 필요 시 롤백할 수 있습니다.
- **A/B 테스트**: 트래픽을 분산하여 신규 모델의 성능을 비교 테스트합니다.
