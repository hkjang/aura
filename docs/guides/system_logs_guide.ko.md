# 시스템 로그 (Logs) 가이드

이 문서는 시스템의 주요 이벤트를 기록하고 조회하는 **로그** 기능에 대한 가이드입니다.

**메뉴 위치:** `시스템 > 로그`

---

## 1. 개요

개발자와 관리자가 시스템의 기술적인 문제를 트러블슈팅하기 위해 사용하는 상세 로그 뷰어입니다. 감사(Audit) 로그와 달리 시스템 에러, API 호출 실패, 성능 이슈 등 기술적인 내용이 주를 이룹니다.

---

## 2. 로그 레벨 (Log Levels)

- **INFO**: 정상적인 작동 정보 (예: 서버 시작됨, DB 연결 성공)
- **WARN**: 잠재적인 문제 (예: API 응답이 평소보다 느림, 디스크 공간 부족 경고)
- **ERROR**: 기능이 실패한 심각한 오류 (예: 모델 API 호출 실패 500 Error, 로그인 실패)

---

## 3. 필터 및 검색

- **기간 설정**: 특정 시간대(예: 지난 1시간)의 로그만 조회합니다.
- **키워드 검색**: 에러 메시지의 일부나 Request ID로 특정 로그를 찾습니다.
- **소스 필터**: `Backend`, `Frontend`, `Worker` 등 로그가 발생한 컴포넌트별로 필터링합니다.

---

## 4. 로그 내보내기

장애 분석을 위해 로그 데이터를 파일(`json` 또는 `csv`)로 다운로드하여 외부 분석 도구(Splunk, ELK 등)로 보낼 수 있습니다.
