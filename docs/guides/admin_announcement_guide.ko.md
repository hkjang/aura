# 관리자 공지사항 관리 가이드

이 문서는 **관리자(ADMIN)** 권한을 가진 사용자가 Aura 포털에 공지사항을 게시하고 관리하는 방법을 상세하게 설명합니다.

---

## 1. 개요

관리자는 전체 사용자를 대상으로 중요 공지사항, 시스템 점검 일정, 기능 업데이트 등을 배너 형태로 알릴 수 있습니다. 공지사항은 포털 상단에 배너로 표시되며, 우선순위에 따라 색상이 다르게 표현됩니다.

---

## 2. 공지사항 작성하기

**위치:** 공지 관리 페이지 (현재 메뉴에 미노출, `/api/admin/announcements` API를 통해 관리됨, 또는 차후 활성화 예정 메뉴)

_참고: 현재 UI상 메뉴 위치가 확인되지 않을 경우, 향후 업데이트를 통해 **시스템 > 설정** 또는 **시스템 > 공지** 메뉴에 추가될 수 있습니다. 아래 가이드는 공지 관리 기능의 표준 사용법입니다._

### 작성 항목 및 설명

1. **제목 (Title)**: 공지사항의 핵심 내용을 간결하게 작성합니다.
   - 예: `[점검] 서버 유지보수 안내`, `신규 AI 모델 추가`
2. **내용 (Content)**: 상세 내용을 작성합니다. 배너 공간 제약상 2~3문장 이내로 핵심만 전달하는 것이 좋습니다.
3. **우선순위 (Priority)**: 공지의 중요도에 따라 4단계 중 선택합니다.
   - **LOW (낮음)**: 회색 배너. 단순 정보 제공.
   - **NORMAL (보통)**: 파란색 배너. 일반적인 업데이트 알림. (기본값)
   - **HIGH (높음)**: 노란색 배너. 주의가 필요한 사항.
   - **CRITICAL (긴급)**: 빨간색 배너. 서비스 장애, 긴급 보안 패치 등 즉시 확인이 필요한 사항.
4. **종료일 (End Date)**: 공지가 자동으로 내려갈 날짜를 지정합니다. 선택하지 않으면 수동으로 내릴 때까지 계속 표시됩니다.
5. **활성화 (Active)**: 체크 시 즉시 사용자에게 노출됩니다.

---

## 3. 공지사항 노출 방식 (배너)

작성된 공지사항은 모든 페이지의 최상단 헤더 아래에 배너 형태로 나타납니다.

### 사용자 상호작용

- **다중 공지**: 활성화된 공지가 여러 개인 경우 슬라이드 형태로 표시되거나, 사용자가 인디케이터를 통해 넘겨볼 수 있습니다.
- **닫기**: 사용자는 `X` 버튼을 눌러 개별 공지를 닫을 수 있습니다. 닫은 공지는 브라우저 세션 동안 다시 나타나지 않습니다.

---

## 4. 관리 및 운영 팁

### 효과적인 공지 운영

- **긴급도 구분**: 항상 `CRITICAL`(빨간색)을 사용하면 사용자가 피로감을 느껴 진짜 중요한 공지를 놓칠 수 있습니다. 평소에는 `NORMAL`(파란색)을 사용하세요.
- **기간 설정**: 종료일(End Date)을 반드시 설정하여, 이미 지나간 이벤트(예: 지난주 점검 공지)가 계속 떠 있는 것을 방지하세요.
- **간결함**: 배너는 공간이 좁으므로, 자세한 내용은 별도 문서 링크를 제공하거나 짧게 요약하여 전달하세요.
