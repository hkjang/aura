# 품질 분석 (Quality Analysis) 가이드

이 문서는 AI 모델의 응답 품질을 지속적으로 모니터링하고 개선하기 위한 **품질 분석** 기능에 대해 설명합니다.

**메뉴 위치:** `분석 > 품질`

---

## 1. 개요

단순히 AI를 도입하는 것을 넘어, AI가 "얼마나 잘 답변하고 있는지"를 정량적으로 평가하고 관리하는 것이 중요합니다. 품질 분석 대시보드는 사용자 피드백과 자동 평가 지표를 종합하여 보여줍니다.

---

## 2. 주요 지표 (Metrics)

### 2.1 사용자 피드백 (User Feedback)

- **좋아요/싫어요 비율**: 사용자가 답변에 대해 긍정/부정 평가를 내린 비율입니다.
- **피드백 내용**: "싫어요"를 누른 경우 사용자가 남긴 구체적인 불만 사항(예: "정보가 틀림", "너무 장황함")을 분석합니다.

### 2.2 응답 관련 지표

- **평균 응답 길이**: 모델이 생성한 답변의 평균 토큰 수입니다.
- **재질문 비율**: 사용자가 첫 답변에 만족하지 못하고 다시 질문한 비율을 추적하여, 답변의 완결성을 평가합니다.

---

## 3. 품질 개선 프로세스

1. **문제 식별**: '싫어요' 비율이 높은 특정 주제나 모델을 식별합니다.
2. **원인 분석**: 피드백 상세 내용을 검토하여 원인(예: 지식 베이스 부족, 프롬프트 오류)을 파악합니다.
3. **개선 조치**:
   - 지식 베이스에 관련 문서 추가
   - 시스템 프롬프트 수정 (더 친절하게, 또는 더 간결하게 등)
   - 모델 변경 (더 성능 좋은 모델로 교체)
