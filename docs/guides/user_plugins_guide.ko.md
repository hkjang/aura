# 플러그인 (Plugins) 가이드

이 문서는 Aura 포털의 **플러그인** 기능을 사용하여 AI가 단순 텍스트 생성을 넘어 외부 시스템과 상호작용하고 실질적인 작업을 수행하도록 확장하는 방법을 설명합니다.

**메뉴 위치:** `관리 > 플러그인`

---

## 1. 개요

플러그인은 AI에게 "손과 귀"를 달아주는 것과 같습니다. 인터넷 검색, 사내 데이터베이스 조회, 메일 발송 등 AI가 자체적으로 할 수 없는 일들을 외부 도구와 연동하여 가능하게 합니다.

### 왜 필요한가요?

- **최신 정보 접근**: LLM은 학습 시점 이후의 정보를 모르지만, '검색 플러그인'을 통해 실시간 뉴스에 접근할 수 있습니다.
- **사내 시스템 연동**: 'Jira 플러그인'으로 티켓을 생성하거나, 'DB 조회 플러그인'으로 재고 현황을 파악할 수 있습니다.

---

## 2. 플러그인 관리 및 설정

### 2.1 플러그인 목록 확인

관리자 메뉴의 플러그인 페이지에서 현재 사용 가능한 도구 목록을 볼 수 있습니다.

- **기본 제공 (Built-in)**: 웹 브라우징, 계산기 등
- **커스텀 (Custom)**: API 연동을 통해 직접 등록한 도구

### 2.2 활성화 및 설정

1. 사용하려는 플러그인을 찾아 **[설정]** 버튼을 누릅니다.
2. 필요한 경우 **API Key**나 **인증 토큰**을 입력합니다. (예: Google Search API Key)
3. **활성화 스위치(Toggle)**를 켜서 사용 가능 상태로 만듭니다.

---

## 3. 사용 방법 (채팅)

플러그인이 활성화되면 채팅이나 에이전트 모드에서 AI가 필요할 때 자동으로 도구를 호출합니다.

### 시나리오 예시

1. **사용자**: "삼성전자의 오늘 주가와 관련 최신 뉴스를 알려줘."
2. **AI**: (자체 지식으로 답변 불가 판단) -> **웹 검색 플러그인 호출**
3. **AI**: 검색 결과를 바탕으로 요약 답변 생성

### 명시적 사용

일부 모델이나 설정에서는 사용자가 "검색 도구를 써서 찾아줘"라고 명시해야 더 잘 작동할 수 있습니다.

---

## 4. 커스텀 플러그인 개발 (고급)

개발자는 REST API 명세(OpenAPI Spec/Swagger)를 등록하여 자체 시스템을 플러그인으로 추가할 수 있습니다.

1. **Manifest 파일 작성**: 플러그인의 이름, 설명, API 엔드포인트를 정의합니다.
2. **등록**: 플러그인 관리 페이지에서 'Custom Plugin 추가'를 선택하고 Spec을 입력합니다.
3. **테스트**: 채팅창에서 플러그인이 의도대로 API를 호출하는지 확인합니다.
