# 시스템 설정 (Settings) 가이드

이 문서는 Aura 포털의 동작 방식을 전역적으로 제어하는 **설정** 메뉴에 대한 심화 가이드입니다.

**메뉴 위치:** `시스템 > 설정`

---

## 1. 일반 설정 (General)

관리자 권한이 있는 사용자만 접근 가능합니다.

### 1.1 브랜딩 및 UI 커스터마이징

- **사이트 이름/파비콘**: 브라우저 탭에 표시될 정보를 수정합니다.
- **테마 설정**: 다크 모드/라이트 모드 기본값을 지정하거나, 회사 브랜드 컬러(Primary Color Hex Code)를 입력하여 UI 전반의 색조를 변경할 수 있습니다.
- **공지사항 배너**: 로그인 화면이나 대시보드 상단에 전사 공지를 띄울 수 있습니다.

---

## 2. 모델 연결 (Model Connections)

### 2.1 클라우드 모델 (OpenAI, Anthropic, Google)

API 키를 안전하게 저장합니다. 키는 DB에 암호화되어 저장되며, UI에서는 마스킹 처리됩니다.

- **Org ID**: 기업용 계정의 경우 Organization ID를 입력해야 정확한 과금 집계가 가능합니다.
- **Rate Limit**: API 호출 빈도 제한을 설정하여 예상치 못한 요금 폭탄을 방지합니다.

### 2.2 로컬/프라이빗 모델 (Ollama, vLLM, Azure)

사내 GPU 서버나 Azure OpenAI Service와 연동합니다.

- **Base URL**: `https://your-azure-endpoint.com/openai` 와 같은 엔드포인트 주소.
- **API Version**: Azure의 경우 `2023-05-15` 등의 버전 지정이 필요할 수 있습니다.
- **Custom Header**: 게이트웨이 인증이 필요한 경우 추가 헤더를 설정할 수 있습니다.

---

## 3. 사용자 인증 및 보안 (Auth & Security)

### 3.1 로그인 방식

- **Credentials**: 이메일/비밀번호 방식 (기본).
- **SSO (Single Sign-On)**: Google Workspace, Okta, Active Directory 연동을 지원합니다. (엔터프라이즈 라이선스 필요)

### 3.2 세션 정책

- **Idle Timeout**: 보안을 위해 사용자가 30분(설정 가능) 이상 활동이 없으면 자동 로그아웃 시킵니다.
- **최대 동시 접속**: 한 계정으로 여러 기기에서 동시 접속하는 것을 제한할 수 있습니다.

---

## 4. 데이터 보존 (Data Retention)

- **채팅 이력 보관 기간**: 기본적으로 무기한 보관되지만, 개인정보 보호 정책에 따라 "90일 후 자동 삭제" 등으로 설정할 수 있습니다.
- **로그 레벨**: `DEBUG` 모드를 켜면 상세한 시스템 로그가 남지만 디스크 용량을 많이 차지하므로, 평소에는 `INFO` 모드를 권장합니다.
