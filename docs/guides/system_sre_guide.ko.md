# SRE (Site Reliability Engineering) 가이드

이 문서는 시스템의 안정성과 성능을 모니터링하는 **SRE** 기능에 대한 가이드입니다.

**메뉴 위치:** `시스템 > SRE`

---

## 1. 개요

Aura 포털이 중단 없이 안정적으로 운영되고 있는지 실시간으로 감시하는 대시보드입니다. 시스템 관리자와 DevOps 엔지니어를 위한 기능입니다.

---

## 2. 모니터링 항목

### 2.1 시스템 상태 (System Health)

- **가동 시간 (Uptime)**: 시스템이 중단 없이 운영된 시간 비율(예: 99.9%)을 표시합니다.
- **CPU/Memory 사용률**: 서버의 리소스 사용 현황을 그래프로 보여줍니다.

### 2.2 성능 지표 (Performance Metrics)

- **평균 응답 시간 (Latency)**: 사용자의 요청부터 응답까지 걸리는 평균 시간입니다. 갑자기 높아지면 장애의 전조일 수 있습니다.
- **에러율 (Error Rate)**: 전체 요청 중 실패한 요청(5xx 에러)의 비율입니다.

### 2.3 데이터베이스 상태

- DB 연결 상태 활성화 여부와 쿼리 응답 속도를 모니터링합니다.

---

## 3. 장애 대응

SRE 대시보드에서 이상 징후(빨간색 경고 등)가 발견되면, 즉시 로그 메뉴로 이동하여 상세 원인을 파악해야 합니다.
