# 에이전트 (Agents) 가이드

이 문서는 Aura 포털의 **AI 에이전트** 기능을 활용하여 복잡한 자율 작업을 수행하는 방법을 설명합니다.

**메뉴 위치:** `AI 사용 > 에이전트`

---

## 1. 개요

일반적인 채팅이 사용자의 질문에 단순히 답하는 것이라면, **에이전트**는 목표를 달성하기 위해 스스로 계획을 수립하고, 도구를 사용하며, 다단계 작업을 수행하는 자율 시스템입니다.

### 주요 특징

- **도구 사용 (Tool Use)**: 웹 검색, 코드 실행, 파일 시스템 접근 등 외부 도구를 활용합니다.
- **계획 수립 (Planning)**: 복잡한 작업을 작은 단위로 나누어 순차적으로 처리합니다.
- **자율성 (Autonomy)**: 중간 결과가 부족하면 스스로 재시도하거나 다른 방법을 찾습니다.

---

## 2. 에이전트 종류

### 2.1 조사 에이전트 (Research Agent)

인터넷을 검색하여 심층적인 정보를 수집하고 정리합니다.

- **용도**: 시장 조사, 경쟁사 분석, 학술 논문 요약
- **사용법**: "2024년 생성형 AI 시장 동향에 대해 조사해서 보고서 써줘"라고 입력.

### 2.2 코딩 에이전트 (Coding Agent)

소프트웨어 개발 작업을 수행합니다. 파일 생성, 수정, 테스트 실행이 가능합니다.

- **용도**: 기능 구현, 버그 수정, 단위 테스트 작성
- **사용법**: "현재 프로젝트의 `auth.ts` 파일을 리팩토링하고 테스트 코드를 만들어줘."

### 2.3 데이터 분석 에이전트 (Data Analyst)

CSV, Excel 파일을 업로드하면 데이터를 분석하고 시각화합니다.

- **용도**: 판매 데이터 분석, 트렌드 예측, 그래프 생성
- **사용법**: 엑셀 파일을 업로드하고 "월별 매출 추이를 꺾은선 그래프로 그려줘"라고 요청.

---

## 3. 워크플로우 설정

### 커스텀 에이전트 만들기

1. **[새 에이전트 생성]** 버튼을 클릭합니다.
2. **역할(Role)** 및 **목표(Goal)**를 설정합니다.
3. 사용할 **도구(Tools)**를 선택합니다. (예: `GoogleSearch`, `Calculator`, `CodeInterpreter`)
4. **저장** 후 바로 사용할 수 있습니다.

---

## 4. 모니터링 및 제어

에이전트가 작업을 수행하는 동안 실시간 로그를 확인할 수 있습니다.

- **생각 과정 (Thought Process)**: 에이전트가 현재 어떤 고민을 하고 있는지 텍스트로 표시됩니다.
- **도구 실행 (Action)**: 어떤 도구를 사용했고 결과가 무엇인지 보여줍니다.
- **개입 (Intervention)**: 작업이 잘못된 방향으로 가면 중지하거나 피드백을 주어 수정할 수 있습니다.
