# 관리 (Manage) 가이드

이 문서는 관리자 및 지식 관리자를 위한 데이터 및 정책 관리 기능에 대한 가이드입니다.

## 1. 지식 베이스 (Knowledge Base)

RAG(검색 증강 생성) 시스템의 핵심인 문서를 관리합니다.

### 문서 인덱싱 절차

1. **업로드**: PDF, Word, txt 등의 문서를 업로드합니다.
2. **청킹(Chunking)**: 시스템이 문서를 검색 가능한 크기의 조각으로 자동 분할합니다.
3. **임베딩(Embedding)**: 텍스트를 벡터로 변환하여 벡터 DB에 저장합니다.
4. **상태 확인**: 상태가 'Indexed'로 변경되면 채팅에서 검색 가능합니다.

### 관리 기능

- **컬렉션**: 문서를 주제별(예: 인사 규정, 기술 문서)로 그룹화하여 관리합니다.
- **검색 및 삭제**: 오래되거나 잘못된 문서를 검색하여 삭제할 수 있습니다.

## 2. 문서 관리 (Documents)

지식 베이스와는 별도로, 포털에서 사용되는 일반적인 파일 자산을 관리합니다.

- 공유 문서, 배포 자료 등을 업로드하고 권한을 설정할 수 있습니다.

## 3. 플러그인 (Plugins)

AI 모델이 외부 도구를 사용할 수 있도록 연결합니다.

### 주요 플러그인 예시

- **웹 검색**: 최신 정보를 검색하기 위한 Google/Bing 검색 연동.
- **데이터베이스 커넥터**: 사내 DB에서 SQL 쿼리를 실행하여 데이터를 조회.
- **API 연동**: Jira, Slack 등 외부 서비스와 연동하여 액션을 수행.

### 설정 방법

- 각 플러그인의 설정 페이지에서 API 키나 인증 정보를 입력하고 '활성화' 스위치를 켭니다.

## 4. 거버넌스 (Governance)

AI 사용에 대한 보안 및 규정 준수 정책을 설정합니다.

### 정책 (Policies)

- **PII 필터링**: 주민등록번호, 전화번호, 이메일 등 개인정보가 포함된 입력/출력을 자동으로 마스킹하거나 차단합니다.
- **금지어 설정**: 비속어, 경쟁사 명칭 등 특정 단어의 사용을 금지합니다.
- **주제 차단**: '정치', '폭력' 등 특정 카테고리의 대화를 원천 차단할 수 있습니다.

### 감사 (Audit)

- 정책 위반 시도가 발생했을 때의 로그를 기록합니다.
- 누가, 언제, 어떤 내용을 입력하여 차단되었는지 확인하고 대응할 수 있습니다.
