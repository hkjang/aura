# 지식 베이스 (Knowledge Base) 가이드

이 문서는 Aura 포털의 **지식 베이스**를 구축하여 AI가 사내 데이터를 기반으로 정확한 답변을 제공하도록(RAG) 만드는 방법을 설명합니다.

**메뉴 위치:** `관리 > 지식 베이스`

---

## 1. 개요

지식 베이스는 AI가 참고할 수 있는 "문서 라이브러리"입니다. 여기에 등록된 문서는 검색 가능한 벡터 형태로 저장되어, 사용자가 질문할 때 AI가 실시간으로 내용을 찾아보고 답변에 활용합니다.

### 주요 기능

- **문서 업로드 및 인덱싱**: 다양한 포맷(PDF, Word 등)을 지원합니다.
- **RAG(검색 증강 생성)**: LLM이 학습하지 못한 사내 비공개 정보를 활용하게 합니다.
- **최신성 유지**: 재학습 없이 문서 업로드만으로 AI의 지식을 업데이트할 수 있습니다.

---

## 2. 문서 관리

### 2.1 문서 업로드

1. **[문서 추가]** 버튼을 클릭합니다.
2. 업로드할 파일을 선택하거나 드래그 앤 드롭합니다.
   - **지원 파일**: PDF, DOCX, TXT, MD (마크다운)
3. **[업로드]**를 눌러 처리를 시작합니다.

### 2.2 인덱싱 프로세스 (자동)

업로드된 파일은 다음 단계를 거쳐 처리됩니다.

1. **텍스트 추출**: 문서에서 글자를 추출합니다.
2. **청킹(Chunking)**: 긴 텍스트를 문맥 단위(예: 500자)로 쪼갭니다.
3. **임베딩(Embedding)**: 각 조각을 AI가 이해할 수 있는 숫자 벡터로 변환하여 벡터 DB에 저장합니다.
4. **완료**: 상태가 `Indexed`로 바뀌면 즉시 검색 가능합니다.

### 2.3 컬렉션 관리

문서를 성격에 따라 폴더처럼 그룹화할 수 있습니다.

- 예: `인사 규정`, `IT 매뉴얼`, `프로젝트 A 기획서`
- 채팅 시 특정 컬렉션만 선택하여 검색 범위를 좁힐 수 있습니다.

---

## 3. 검색 결과 품질 관리

AI가 엉뚱한 문서를 참고한다면 다음을 확인하세요.

### 청킹 전략 수정

문서의 특성에 따라 청크 크기(Chunk Size)와 중복(Overlap) 설정을 조정할 수 있습니다.

- **짧은 문서**: 청크 크기를 줄여서 세밀하게 검색하도록 설정.
- **긴 문맥이 필요한 문서**: 청크 크기를 늘려 문맥이 끊기지 않게 설정.

### 문서 정제

- 이미지 위주의 스캔 문서는 OCR 품질에 따라 인식이 잘 안 될 수 있습니다.
- 가급적 텍스트 복사가 가능한 원본 파일을 사용하는 것이 좋습니다.
