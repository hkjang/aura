# 관리자 (ADMIN) 가이드

이 문서는 Aura 포털의 관리자(ADMIN)를 위한 시스템 운영 및 관리 가이드입니다.

## 1. 초기 설정 점검

시스템을 처음 배포한 후 다음 사항을 점검해야 합니다.

1. **API 키 설정**: `시스템 > 설정` 메뉴에서 OpenAI, Anthropic 등 외부 모델 공급자의 API 키가 올바르게 입력되었는지 확인합니다.
   - [AI 모델 설정 상세 가이드](../guides/admin_ai_model_settings.ko.md) 참조
2. **기본 모델 지정**: 전사적으로 사용할 기본 LLM 모델을 설정합니다. (비용과 성능을 고려하여 선택)
3. **브랜딩**: 로고 및 헤더 색상을 사내 브랜드 가이드에 맞게 조정합니다.

## 2. 사용자 및 권한 관리

### 신규 입사자 계정 생성

1. **[사용자 관리]** 메뉴로 이동합니다.
2. **[사용자 추가]** 버튼을 클릭합니다.
3. 이메일, 이름, 초기 비밀번호를 설정합니다.
4. 역할을 `USER`로 지정합니다. (일반 사용자)

### 권한 승격 (부서장/지식 관리자)

특정 사용자에게 지식 베이스 관리 권한을 부여하려면 역할을 조정하거나 별도의 권한 그룹에 추가할 수 있습니다.

## 3. 지식 베이스(RAG) 구축 및 관리

성공적인 RAG 도입을 위해 양질의 데이터를 관리하는 것이 중요합니다.

### 문서 인덱싱 전략

- **전처리**: 문서를 업로드하기 전에 스캔본 등 텍스트 인식이 불가능한 파일은 OCR 처리를 해야 합니다. (현재 버전은 텍스트 추출이 가능한 PDF/Word 지원)
- **청킹(Chunking) 설정**: 문서의 특성에 따라 청크 크기를 조절할 수 있습니다 (고급 설정). 일반적으로 500~1000 토큰이 적당합니다.
- **정기 업데이트**: 규정이나 매뉴얼이 개정되면 구 버전을 삭제하고 새 버전을 업로드해야 합니다.

## 4. 보안 및 거버넌스 설정

### PII(개인정보) 필터링 테스트

1. **[거버넌스 > 정책]** 메뉴에서 PII 필터가 활성화되어 있는지 확인합니다.
2. 테스트 계정으로 채팅에서 가상의 주민등록번호나 전화번호를 입력해봅니다.
3. 시스템이 이를 마스킹 처리(`***-****`)하거나 차단하는지 확인합니다.

### 금지어 등록

경쟁사 명칭, 비속어, 보안상 민감한 프로젝트 코드명 등을 금지어로 등록하여 유출을 방지합니다.

## 5. 비용 및 모니터링 관리

### 비용 알림 설정

1. **[비용 분석]** 메뉴에서 월간 예산 한도를 설정합니다.
2. 예산의 80%, 90%, 100% 도달 시 이메일 알림을 받도록 설정합니다.

### 시스템 장애 대응

1. **[SRE]** 대시보드를 주기적으로 모니터링합니다.
2. 특정 모델의 에러율이 급증하면 **[시스템 > 설정]**에서 해당 모델을 일시적으로 비활성화하고, 다른 모델을 기본값으로 변경하여 사용자 불편을 최소화합니다.

## 6. 감사(Audit) 로그 분석

보안 사고 예방을 위해 주 1회 이상 감사 로그를 검토하는 것을 권장합니다.

- **[감사]** 메뉴에서 `ACCESS_DENIED`, `POLICY_VIOLATION` 등의 이벤트가 빈번한 사용자가 있는지 확인합니다.
